About Sentinel

Built by security engineers, for AI teams.

Our Mission

Unified security for AI agents

AI agents are making autonomous decisions in regulated industries — handling patient records, processing financial transactions, managing pharmaceutical trial data. Every one of these decisions is a potential compliance breach, data leak, or privilege escalation. The tooling to secure them didn't exist as a unified product.

Sentinel was born from five separate open-source repositories, each solving one piece of the AI agent security puzzle. Individually, they worked. Together, they left gaps — different logging formats, no shared dashboard, no unified policy enforcement. We merged them into a single platform where every security layer talks to every other, every event feeds the same dashboard, and one install command protects every agent on the machine.

Designed for regulated industries — fintech, healthcare, and pharma — Sentinel detects 40+ compliance breach types in under 1 millisecond using deterministic rule-based classifiers. No LLM sits in the critical path. Every detection produces an immutable, SHA-256-chained audit record that satisfies PCI-DSS, HIPAA, FDA 21-CFR Part 11, and SOX audit requirements.

The result: a single platform that gives AI teams the confidence to deploy agents in production, knowing that every prompt, every tool call, every LLM response, and every outgoing message is monitored, analyzed, and enforced in real time.

Principles

How we build

⚡

No LLM in the Critical Path

Rule-based detection runs in under 1ms. LLM analysis is always asynchronous and optional — it never blocks the security pipeline or adds latency to agent responses.

🔗

Immutable by Default

Every audit record is SHA-256 checksummed and chained. The database allows INSERT only — no UPDATE, no DELETE. Tamper detection runs automatically on every read.

🔌

Zero Code Changes

Auto-instrumentation via sitecustomize.py means agents are protected the moment they run. No SDK, no decorators, no middleware. Just install and run.
